# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Stack

- **Runtime:** .NET 8 Web API (`BlueberryMart.Api/`)
- **Database:** PostgreSQL via Entity Framework Core 8 + Npgsql; schema managed by **EF Core migrations** (`BlueberryMart.Api/Migrations/`)
- **Tests:** `Tests/BlueberryMart.Api.Tests/`

## Commands

```bash
# Run the API
dotnet run --project BlueberryMart.Api

# Build
dotnet build BlueberryMart.Api

# Run tests
dotnet test Tests/BlueberryMart.Api.Tests

# Add a migration after changing an entity (dotnet-ef is a local tool)
dotnet dotnet-ef migrations add <Name> --project BlueberryMart.Api --output-dir Migrations

# Apply pending migrations to a database manually (usually unnecessary)
dotnet dotnet-ef database update --project BlueberryMart.Api
```

The app calls `context.Database.Migrate()` on startup (`DbInitializer.Initialize`),
so deploying new code **applies pending migrations to the target database
automatically** — no manual step. The legacy raw `.sql` files in
`Database/Migrations/` are kept for history only and are no longer the source of truth.

## Architecture

The API follows a layered architecture inside `BlueberryMart.Api/`:

```
Controllers/        — HTTP entry points; thin, delegate to Services
Services/           — business logic
  Interfaces/       — service contracts
Repositories/       — data access (raw SQL via Npgsql or Dapper)
  Interfaces/       — repository contracts
Models/
  Entities/         — DB row representations
  DTOs/             — internal data transfer objects
  Requests/         — deserialized request bodies
  Responses/        — serialized response bodies
Middleware/         — cross-cutting concerns (auth, error handling, logging)
Extensions/         — IServiceCollection / IApplicationBuilder extension methods
Configuration/      — strongly-typed settings bound from appsettings.json
```

## Database conventions

- Schema changes are made by editing entities in `Models/Entities/` then running `dotnet ef migrations add`. Migrations apply automatically on app startup.
- All tables use `UUID` primary keys generated by `gen_random_uuid()`.
- Timestamps are `TIMESTAMPTZ` (UTC).
- Domain enums (`user_role`, `order_type`, `order_status`) live as PostgreSQL `ENUM` types defined in `01_InitSchema.sql`.
- Foreign keys use `ON DELETE RESTRICT` by default.
